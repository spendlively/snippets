//////////////////////////
//Расположение библиотек//
//////////////////////////
 - динамические библиотеки загружаются утилитами ld.so (ld.so.N) и ld-linux.so (ld-linux.so.N)
 - в линуксе /lib/ld-linux.so.x ищет и загружает все используемые динамические библиотеки

 - приложение может вызвать библиотеку
    - по имени библиотеки или
    - по абсолютному имени файла

 - по умолчанию библиотеки хранятся в
    - /usr/local/lib
    - /usr/local/lib64
    - /usr/lib
    - /usr/lib64
 - системные библиотеки хранятся в
    - /lib
    - /lib64
 - разработчики однако могут устанавливать библиотеки в любых местах

//////////////////////////////////
//Конфигурация пути к библиотеке//
//////////////////////////////////
//Статическая
 - путь к библиотеке может быть прописан в /etc/ld.so.conf (/etc/ld.so.conf.d/*.conf)
 - разработчики могут добавлять свои кастомные библиотеки в /etc/ld.so.conf.d/*.conf

//Динамическая
 - для динамической конфигурации пути используется переменная окружения LD_LIBRARY_PATH
 - export LD_LIBRARY_PATH=/path/to/library/file
 - можно конфигурировать LD_LIBRARY_PATH при запуске ОС
 - /etc/profile (global) or ~/.profile (user-specific)

/////////////////////////////////////////
//Управление динамическими библиотеками//
/////////////////////////////////////////
//Получение списка всех динамических библиотек слинкованых с конкретным бинарником
 - ldd /usr/bin/ls

//Кэширование путей динамических библиотек
 - чтобы сократить поиск библиотек они кэшируются с помощью утилиты ldconfig
 - порядок работы ldconfig
    - ldconfig читает содержимое /etc/ld.so.conf
    - создает симлинки в стандартных директориях для дин. библиотек
    - сохраняет кэш в /etc/ld.so.cache
 - после установки новой динамической библиотеки, создания собственной библиотеки
или создания новой директории для дин. библиотек нужно вызвать ldconfig чтобы применить изменения
 - sudo ldconfig -v
 - после создания новой библиотеки ее нужно установить в стандартную директорию
 - или запустить ldconfig
 - sudo ldconfig -v
 - sudo ldconfig -n /path/to/your/shared/libraries
