

//Для начальной установки объявленных зависимостей:
php composer.phar update

//Что произойдет
1. Шаг 1 (composer update)
 - резолвит все зависимости перечисленные в composer.json
 - записывает все пакеты и их конкретные версии в composer.lock
 - лочит проект с этими конкретными версиями
 - вы должны коммитить composer.lock в репозиторий, чтобы
все работали с одними и теме же версиями зависимостей

2. Шаг 2 (composer install)
 - неявно вызывает команду install
 - install скачивает файлы зависимостей в директорию vendor
 - например пакет monolog/monolog скачается в vendor/monolog/monolog/
 - т.к. пакет Monolog имеет свои зависимости от psr/log, файлы этого пакета также попадут в vendor/
