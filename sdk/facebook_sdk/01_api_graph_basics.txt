////////////Основы/////////////
///////////////////////////////
//узлы (nodes)
 - отдельные "объекты" (например, Пользователь, Фото, Страница или Комментарий);
 - С помощью узлов можно получать данные о конкретном объекте
 - узлы позволяют собирать данные об отдельных объектах
 - Узлы — это отдельные объекты, у каждого из которых есть уникальный ID
 - чтобы получить информацию о каком-то узле, необходимо запросить его ID
 - У большинства узлов есть границы контекста, с помощью которых можно получить подборки объектов, связанных с этим узлом.
Чтобы запросить границу контекста, следует указать как ID узла, так и его имя
https://graph.facebook.com/20531316728/photos?access_token={access-token}
все объекты фото, принадлежащие Странице:
https://graph.facebook.com/20531316728/photos?access_token={access-token}
 - Некоторые узлы позволяют обновлять поля с помощью операций POST
curl -i -X POST "https://graph.facebook.com/20531316728?description=The%20OFFICIAL%20Facebook%20Page&access_token={access-token}"

//границы контекста (edges)
 - связи между подборкой объектов и отдельным объектом,
например фото на Странице или комментарии к фото;
 - границы контекста позволяют собирать подборки объектов, связанные с отдельным объектом
 - границы контекста помогают собирать подборки объектов, связанных с узлом, или публиковать объекты в этих подборках
 - с помощью границ контекста можно публиковать новые объекты в подборках узла
Опубликовать фото в подборке фото, принадлежащих Странице Facebook
curl -i -X POST "https://graph.facebook.com/20531316728/photos?access_token={access-token}"

//поля (fields)
 - данные об объекте, например день рождения пользователя или название Страницы
 - поля помогают получать данные об отдельном объекте или о каждом объекте в подборке
 - с помощью полей можно указать данные, которые вы хотите включить в отклики
 - Если нужно получить конкретные данные об узле (поля),
включите параметр fields и укажите, какие поля должны быть в отклике
https://graph.facebook.com/20531316728?fields=cover&access_token={access-token}




///////////////////////////////
//////////HTTP запросы/////////
///////////////////////////////
 - API Graph основан на HTTP

//В браузере
https://graph.facebook.com/facebook/picture?redirect=false

//CURL
curl -i -X GET "https://graph.facebook.com/facebook/picture?redirect=false&access_token={valid-access-token-goes-here}"

//Адрес сервера
 - почти все запросы передаются в URL хоста graph.facebook.com
 - исключением являются видео, которые используют для этого graph-video.facebook.com

//как отправить вызов в определенную версию
curl -i -X GET "https://graph.facebook.com/v2.9/20531316728/photos?access_token={access-token}"
