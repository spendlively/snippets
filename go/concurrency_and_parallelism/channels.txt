

//Каналы

//Блокировки


//При записи
 - Когда горутина посылает данные в канал,
она блокируется до тех пор, пока другая горутина не прочитает данные из канала



//При чтении
 - горутина может быть заблокирована при чтении из канала до тех пор,
пока в него ничего не запишут

Блокировки можно представить как «передачу денег» между двумя горутинами.
Когда один гофер хочет передать или получить деньги,
то ему приходится ждать второго участника сделки.



//Небуферизованные каналы
 - можно передать только один фрагмент данных за раз



//Буферизованные каналы
 - В буферизоваванный канал можно послать несколько фрагментов данных,
без необходимости чтения их другой горутиной
 - для канала емкостью 2, отправляющая горутина будет каждый раз заблокирована
при отправке, если в канале уже есть два фрагмента данных.
