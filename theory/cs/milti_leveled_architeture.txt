///////////////////////////////////////////////////////////
///////////////////////Устройство//////////////////////////
///////////////////////////////////////////////////////////
Компьютер можно прелставить как иерархическую систему уровней, каждый из которых:
 - ПРЕДСТАВЛЯЕТ УРОВЕНЬ АБСТРАКЦИИ НАД ПРЕДЫДУЩИМ УРОВНЕМ
 - ХАРАКТЕРИЗАЕТСЯ своим ЯЗЫКОМ, различными ОБЪЕКТАМИ и ОПЕРАЦИМИ над ними

Большинство современных компьютеров состоит из двух и более (до 6ти) уровней.



///////////////////////////////////////////////////////////
/////////////////////Список уровней////////////////////////
///////////////////////////////////////////////////////////
1. Цифровой логический уровень
2. Уровень микроархитектуры
3. Уровень архитектуры комманд
4. Уровень операционной системы
5. Уровень языка ассемблера
6. Язык высокого уровня

Ниже ЦЛУ существует физический уровень, на котором находятся транзисторы,
он не попадает в сферу электронной техники поэтому не рассматривается.

Первые три уровня конструируются не для того,
чтобы с ними работал обычный программист

Уровни с четвертого и выше предназначены для прикладных программистов, решающих конкретные задачи.

Уровни 3 и 4 обычно интерпретируются, а уровни 5, 6 и выше обычно,
хотя и не всегда, поддерживаются транслятором.

Машинные языки уровней 2, 3 и 4 — цифровые. Программы, написанные на
этих языках, состоят из длинных рядов цифр, которые удобны для компьютеров,
но совершенно неудобны для людей. Начиная с четвертого уровня, языки содержат слова и сокращения, понятные человеку



///////////////////////////////////////////////////////////
/////////////////Описание работы уровней///////////////////
///////////////////////////////////////////////////////////
1. Цифровой логический уровень
 - язык - цифровые сигналы
 - объекты - вентили
 - операции - операции И, ИЛИ, НЕ, ...

На цифровом логическом уровне, объекты называются вентилями,
каждый из которых формируется из нескольких транзисторов.

У каждого вентиля есть одно или несколько цифровых входных данных
(сигналов, представляющих 0 или 1).

Вентиль вычисляет простые функции этих сигналов, такие как И или ИЛИ.

Несколько вентилей формируют 1 бит памяти, который может содержать 0 или 1.

Биты памяти, объединенные в группы, формируют регистры,
каждый из которых может содержать одно двоичное число до определенного предела.



2. Уровень микроархитектуры
 - язык - числа
 - объекты - регистры формирующие память и АЛУ
 - операции - простые арифметические операции

На этом уровне можно видеть совокупности 8 или 32 регистров,
которые формируют локальную память и схему, называемую АЛУ (арифметико-логическое устройство).

АЛУ выполняет простые арифметические операции.

Регистры вместе с АЛУ формируют тракт данных, по которому поступают данные.

Основная операция тракта данных состоит в следующем:
 - выбирается один или два регистра,
 - АЛУ производит над ними какую-либо операцию, например сложения
 - результат помещается в один из этих регистров



3. Уровень архитектуры комманд
 - язык - машинный код
 - объекты - данные
 - операции - операции процессора

Содержит набор машинных команд.

Каждый производитель публикует руководство для компьютеров, которые он продает,
под названием «Руководство по машинному языку» и т. п.

Такие руководства содержат информацию именно об этом уровне.



4. Уровень операционной системы
 - язык - машинный код
 - объекты - данные
 - операции - операции процессора и операционной системы

У этого уровня есть некоторые дополнительные особенности:
 - набор новых команд
 - другая организация памяти
 - способность выполнять две и более программ одновременно
 - и др.

Одна часть команд этого уровня интерпретируется операционной системой,
а другая часть — микропрограммой.

Поэтому этот уровень считается гибридным.



5. Уровень языка ассемблера
 - язык - азык ассемблера
 - объекты - данные
 - операции - операции поддерживаемые языком ассемблера

Уровень представляет собой символическую форму одного из языков более низкого уровня.

На этом уровне можно писать программы в приемлемой для человека форме.

Эти программы сначала транслируются на язык уровня 2, 3 или 4,
а затем интерпретируются соответствующей виртуальной или фактически существующей машиной.

Программа, которая выполняет трансляцию, называется ассемблером



6. Уровень языка высокого уровня
 - язык - язык программирования
 - объекты - данные
 - операции - операции яп

Состоит из языков, разработанных для прикладных программистов.

Такие языки называются языками высокого уровня
